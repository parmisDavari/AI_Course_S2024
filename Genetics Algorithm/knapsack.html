Genetics Algorithm: Knapsack Problem¶

Problem Description:¶

Assuming we have a backpack, we have to fill this backpack with a number of items each with certain weights and values. The goal in thus problem is to use certain amount of items that don't exceed the weight limit of the problem. Also, the items and their amounts should be chosen in a way that do exceed the value limit of the problem. Also, there is another constraint in the problem, we can only choose items of variety in a certain numeric range. We want to solve this problem with these constraints:

  1. Weight Limit = 10
  2. Minimum Value = 12
  3. Items in variety chosen in range (2,4)

Goal:Maximize the total value of selected snacks while adhering to the specified constraints.¶

After making the whole population, we have to define a fitness function in order to identify the best solutions. According to the knapsack, the fitness function should be in a way that chooses the solutions that maximize the total value of the backpack, and also doesn't exceed the weight limit.

$$ value(chromosome) = \sum_{i=1}^{N} v_i \cdot x_i \geq V $$

$$ weight(chromosome) = \sum_{i=1}^{N} w_i \cdot x_i \leq W $$

$$ 2 \leq count Of Snacks \leq 4 $$

$$ weight Punishment(chromosome) = (weight > 10)(weight - 10) * c1 $$

$$ num Punishment(chromosome) =(num > 4) * (num - 4) (num < 2) * (2 - num) $$

In [ ]:
def punishment(chromosome):
    C1, C2 = 1, 1
    weight = (data_frame["Available Weight"].to_numpy() * chromosome).sum()
    weight_punishment = (weight>10)*(weight - 10) * C1
    num = np.count_nonzero(chromosome)
    num_punishment = ((num>4)*(num-4)+(num<2)*(2-num)) * C2
    total_punishment = weight_punishment + num_punishment 
    return total_punishment

def calc_fitness(chromosome):    
    
    total_value = (chromosome * data_frame['Value'].to_numpy()).sum()
    fitness = total_value - punishment(chromosome)
    return fitness

Now we want to implement the Crossover phase. Which means that we take 2 chromosomes and split them at a certain position, to generate a new child chromosome. Crossover has many types. I am using one-point-crossover which chooses a random number in range of the length of the chromosome and then, splits the chromosomes at that point to create child1 and child2.

In [ ]:
def crossover_one_point(parents, p_crossover = 0) -> tuple:
    if np.random.random(1) > p_crossover:
        return parents
    par1, par2 = parents
    crossover_point = np.random.randint(NUM_OF_SNACKS) 
    child1 = np.concatenate((par1[crossover_point:], par2[:crossover_point]))
    child2 = np.concatenate((par2[crossover_point:], par1[:crossover_point]))
    return normalize_chromosome(child1), normalize_chromosome(child2)

We can also use uniform crossover. In uniform crossover for each gene we decide to give which parent's gene to the first child and the second child will be the inverse of the first child.

def uniform_crossover(parents, p_crossover):
    if np.random.random() > p_crossover:
        return parents
    
    par1, par2 = parents
    child1, child2 = np.empty(shape = (NUM_OF_SNACKS)), np.empty(shape = (NUM_OF_SNACKS))
    for i in range(len(par1)):
        if np.random.bit_generator.randbits(1):
            child1[i] = par1[i]
            child2[i] = par2[i] 
        else:
            child1[i] = par2[i]
            child2[i] = par1[i]   
               
    return normalize_chromosome(child1), normalize_chromosome(child2)

def mutation(chromosome, p_mutation):

from copy import deepcopy
def find_optimal_solution(constants):
    population = gen_initial_population(constants.population_size)
    generation_num = 0
    best_chroms = []
    while generation_num < constants.max_generations:
        fitnesses = np.apply_along_axis(calc_fitness, axis = 1, arr = population)
        best_chroms.append(fitnesses.max())
        sorted_indices = np.argsort(fitnesses)
        population = population[sorted_indices]
        indices = np.arange(0, constants.population_size, 1)
        prob = indices / indices.sum()
        children = np.empty(shape = (constants.population_size, NUM_OF_SNACKS))
        for i in range((constants.population_size) // 2):
            parents = np.random.choice(
                indices,
                2,
                p = prob
            )
            papa = population[parents[0]]
            mama = population[parents[1]]
            childs = crossover_one_point((papa, mama), 0.9)
            first_child = mutation(childs[0], 0.1)
            second_child = mutation(childs[1], 0.1)
            
            children[i] = first_child
            children[(constants.population_size) // 2 + i] = second_child
        
        population = deepcopy(children)
        generation_num += 1
        
    return population, generation_num, best_chroms

import time
def run_genetic(constants):
    print(f" constants: max_generations: {constants.max_generations} population_length: {constants.population_size} p_crossover: {constants.p_crossover} p_mutation: {constants.p_mutation}")
    sum_num_of_generations = 0
    sum_time = 0
    REPEAT = 3
    for i in range(REPEAT):
        t0 = time.time()
        answer, num_of_generations, max_fitnesses = find_optimal_solution(constants)
        if type(answer) == type(None):
            print("Couldn't find answer")
            return 
        t1 = time.time()
        sum_time += (t1 - t0)
        sum_num_of_generations += num_of_generations
    print(f"On average took {sum_num_of_generations / REPEAT} generations and took {sum_time / REPEAT} seconds.")
    return sum_num_of_generations, max_fitnesses

Questions¶

  1. What happens if the population is too big or too small? If the population size is too small, we have less diversity in finding the optimal solution. If the population is too big, then we have a better chance in finding the optimal solution because we have more diversity but it takes more time and space.

  2. What happens if population increases after each generation? Growth of population results in more diversity and better chance of finding more optimal solutions, however the issue of time and space remains. With population growth, it takes more and more time and space and it may not be very efficient. Also, growth of population in each generation, may distrupt the convergence.

  3. What are the effects of crossover and mutation and what happens if we use only one of them? If we skip the Mutation phase, our solutions may only find a local optimum, and don't explore new and better optimal solutions. Mutation phase causes our chromosomes to explore around and they may find a better and more optimal solution. Crossover makes better chromosomes by combining good chromosomes but mutation is mostly used to make better chromosomes by directly changing the genes of one chromosome and helps us escape a local extremum. For example if we check the best fitness in the population of each generation, we can see that it converges to a number and then a mutation causes it to diverge from that number and suddenly increase. So we can't solve the problem by only using crossover.(uniform crossover may reach the answer but it will take too many generations) and in general only using mutation won't lead us to an optimal answer because it doesn't have the effect of crossover, however in this case for some constants, only using mutation leads to answer.
